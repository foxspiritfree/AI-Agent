# Daily Review Reference

Use this reference when the user wants a reflective daily review, not only a work log.

## Reflection Lenses

Use one or two lenses only; do not turn daily review into a long interview.

| Lens | Prompt | Source Cards |
|---|---|---|
| Emotion marker | What emotion, pressure, or stuck point is worth naming today? How strong was it? | `H2B`, `DU1` |
| Meaning from difficulty | What meaning, lesson, or controllable next move can be taken from the hard part? | `F6A2A1A5`, `F6A2A1A4` |
| Learning extraction | What did you understand today, and where can you retrieve or practice it later? | `DZ12B`, `DZ1A1` |
| Small success | What small win happened, even if it looked minor? | `M`, `E8B` |
| Repeated pattern | What repeated situation, trigger, or friction showed up again? | `A1A1B2A4A1A`, `A1A1A1A1A6` |
| Failure precheck | What could make tomorrow's plan fail? | `A1B3B`, `H1A2B` |
| Concrete next step | What is the smallest specific action with time or context attached? | `H4A1`, `E5B` |

## Question Patterns

- Emotion: "今天有沒有一個情緒或壓力點值得標記？如果 1-10 分，大概幾分？"
- Meaning: "這件卡住的事，有沒有一個可以帶走的意義、教訓或下一步？"
- Learning: "今天學到的東西，之後要在哪個情境提取或練習？"
- Small success: "今天有沒有一個很小但值得保留的成功？"
- Pattern: "這是單次事件，還是最近反覆出現的模式？"
- Failure precheck: "明天這件事最可能失敗在哪裡？先避開哪一點？"
- Next step: "明天最小的一步是什麼？要在什麼時間或情境開始？"

## Writing Rules

- Keep the output concrete and searchable.
- Preserve the user's own words for emotions, people, projects, errors, links, and filenames.
- Do not diagnose the user's mental state.
- Convert broad intentions into small actions.
- If the user is tired or vague, ask for one concrete event first.
