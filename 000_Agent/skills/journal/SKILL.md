# Journal

把當天狀態整理成可回查的 Markdown 日記。回答使用繁體中文，語氣直接、簡短，不做心理評估，不主動延伸話題。

## 檔案位置

日記存到：

`300_Journal/YYYY-MM/YYYY-MM-DD.md`

例如：

`300_Journal/2026-05/2026-05-07.md`

如果月份資料夾不存在，先建立。

## 流程

### Step 0：確認日期與現有日記

1. 用 PowerShell 確認今天日期。
2. 檢查今日檔案是否存在。
3. 如果已存在，預設「補充到原日記下方」，不要覆蓋；除非使用者明確說重寫。
4. 如果昨天日記存在，讀取「明天要做的事」與「待追蹤」；只用一句話提醒。

### Step 1：收集內容

如果使用者只說「寫日記」，逐題問，最多 5 題，每題等回答後再問下一題。使用者說「跳過」就跳過。

1. 今天做了哪些事？
2. 有沒有卡住、處理中、需要交接的事？
3. 今天學到或確認了什麼？
4. 明天最重要的 1 到 3 件事？
5. 有沒有待追蹤風險、連結、檔案或指標？

如果使用者已經貼了一段內容，直接整理，不要再從第一題問起；缺少明日待辦時再問一句。

### Step 1B：每日復盤引導

如果使用者說「復盤」「每日復盤」「今天結束了」或明顯想整理狀態，用更有反思感但仍簡短的問法。不要一次丟完整問卷；每次只問下一個最必要問題。

**理想自我介入 (Aspirational Intervention)**：
在引導深度復盤或總結今日時，AI 應主動檢索 `400_Knowledge/self_brain/`（特別是 `identity/` 與 `concepts/` 目錄）中帶有 `confidence: aspirational` 的條目。
根據使用者今天的經歷或情緒，挑選 1 個最相關的「理想方向」，輕輕地給予反饋或提問。
例如：「今天似乎因為某個溝通卡住了。如果套用你正在練習的『搭橋提問』或『微量認可』，這個僵局會有不同的解法嗎？」

優先問：

1. 今天最值得記下來的一件事是什麼？
2. 今天有哪個情緒、壓力或卡點值得標記？強度大概多少？
3. 今天學到或確認了什麼？之後可以怎麼提取或練習？
4. 今天有哪些小成功、微小風險或重複出現的模式？
5. 明天最小、具體、可執行的一步是什麼？

如果使用者提到情緒、壓力、挫折、失敗、學習或習慣，再讀 [daily-review.md](references/daily-review.md) 取用對應引導。

### Step 2：生成格式

```md
# YYYY-MM-DD 日記

## 今天做了什麼
- ...

## 卡住 / 交接
- ...

## 學到 / 確認
- ...

## 情緒 / 模式
- ...

## 明天要做的事
- [ ] ...

## 待追蹤
- ...
```

規則：

- 沒內容的區塊省略。
- 保留使用者給的具體名詞、檔案、指令、錯誤訊息、連結。
- SOC / SecOps 內容要可稽核、可追蹤、可交接。
- 技術或專案內容可以用條列；個人反思保持短段落。
- 情緒內容只做描述與標記，不評估人格、不過度解讀。
- 明日行動要具體到可以開始，優先小步驟而非大願望。
- 不主動問要不要 commit、push 或同步。

### Step 3：存檔

明確收到「寫日記」且內容足夠時，先產生日記內容並存檔。存檔後只回報檔案路徑與簡短驗證方式。

如果內容不足以寫成日記，問下一個最必要的問題，不要一次丟很多問題。

## 驗證

存檔後可用：

```powershell
Get-Content 300_Journal\YYYY-MM\YYYY-MM-DD.md
```
