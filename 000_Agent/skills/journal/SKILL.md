# Journal

把當天狀態整理成可回查的 Markdown 日記。回答使用繁體中文，語氣直接，不做心理評估，不主動延伸話題。

**核心原則**：每次只問一題，等使用者回答後再繼續。使用者說「跳過」就跳過。

## 檔案位置

`300_Journal/YYYY-MM/YYYY-MM-DD.md`

若月份資料夾不存在，先建立。

---

## 流程

### Step 0：確認日期

1. 用 PowerShell 確認今天日期。
2. 檢查今日檔案是否存在。若已存在，預設補充到原日記下方，不覆蓋；除非使用者明確說重寫。

---

### Step 1：逐題引導復盤

依序問以下問題，每次只問一題，等回答後再繼續。使用者說「跳過」則跳過。

#### 暖場（任選一，視當下對話氛圍）
- 「今天最讓你有感的一件事是什麼？不管是好的還是壞的。」
- 「如果用一個詞描述今天的自己，你會選什麼？」
- 「今天什麼事情讓你停下來想了一下？」

---

#### 1｜情緒覺察（必問）

直接問（情緒輪與問題合為一題，不要分兩步）：

「今天最明顯的情緒是什麼？從下面選一個最接近的，再告訴我大概幾分（1-10）：

- **憤怒 / 挫折**：挫折、不滿、憤怒、敵意、怨恨、不公
- **恐懼 / 焦慮**：焦慮、超載、擔憂、無助、緊繃、心慌
- **悲傷 / 失落**：失落、疲憊、失望、沮喪、孤單、愧疚
- **厭惡 / 抗拒**：厭煩、排斥、不屑、反感、抗拒
- **驚訝 / 混亂**：困惑、震驚、迷茫、不知所措
- **喜悅 / 滿足**：滿足、釋懷、平靜、自豪、興奮、愉悅

如果今天沒有特別的情緒，也可以直接說。」

確認情緒後，繼續問：

「這個情緒底下，有什麼『期待』沒有被滿足？或是什麼『恐懼』被觸動了？」

「這個感受在你身體哪裡？是緊繃、沉重，還是什麼感覺？」

（視情況）「這個反應，是當下的情況如此，還是被過去某個記憶或習慣觸發的？」

---

#### 2｜行為模式（視情況問）

「今天有沒有注意到自己有什麼『自動導航』的反應——就是沒有多想就做了的事？」

若有：「這個反應幫了你，還是讓你後來覺得可以不一樣？」

---

#### 3｜自我認知（必問）

「今天有哪個瞬間，你覺得『這是我想要的樣子』或『這不是我想要的樣子』？」

**理想自我連結**：讀取 `400_Knowledge/self_brain/`（特別是 `identity/` 與 `concepts/`）中帶有 `confidence: aspirational` 的條目。
- 從中挑選 1 個與今天情緒或事件**最相關**的練習方向。
- 提取該條目的**具體意義與脈絡**，用對話的語氣自然地包裹在問句中，幫助使用者回憶起這個概念。**不要只講短標籤，也不要生硬地照貼原文指令**。
- 例如：「你之前有在練習『在遇到狀況卡住時，先不歸咎自己，而是切換到系統除錯模式檢查 B=MAP 哪裡出錯』，今天有機會應用到這個概念嗎？」
- 若沒有明顯相關的條目，**跳過此步驟，不問**。

---

#### 4｜人際互動（視情況問）

「今天和別人的互動中，有沒有真正『聽到』對方的感受，而不只是話語？」

「有沒有哪個互動讓你有所啟發，或感到遺憾？」

---

#### 5｜成長視角（必問）

「今天有沒有一件小事，讓你覺得自己多了一點什麼？可以是理解、能力，或只是勇氣。」

「這個學習，未來可以在什麼情境提取？」

---

#### 6｜壓力水位（視情況問）

「今天的壓力水位大概是幾分（1-10）？有沒有什麼在無形中悄悄累積？」

---

### Step 2：生成日記並存檔

收集完內容後，生成以下格式並存檔：

```md
# YYYY-MM-DD 日記

## 今天的脈絡
- （1-3 句背景，供未來回查）

## 情緒覺察
**情緒**：[類型] 強度 X/10
**觸發情境**：
**期待 / 根因**：
**身體感知**：

## 行為模式
- 觸發：
- 我的反應：
- 結果：
- 如果重來：

## 自我認知 × 人際
- 我注意到自己：
- 人際觀察：
- 理想自我落差：

## 成長視角
- 今天學到或確認：
- 可提取的情境：
- 小成功：

## 壓力 / 微壓力
- 水位：X/10
- 來源：
```

**輸出規則**：
- 沒內容的區塊省略。
- 保留使用者給的具體名詞、事件描述、情緒用詞。
- 情緒內容只做描述與標記，不評估人格，不過度解讀。
- 個人反思保持短段落，不拆成過多條列。
- SOC / 技術工作記錄可加在日記最上方，格式不限。
- **不生成明天待辦清單**。

---

### Step 3：卡片行動建議

**存檔後必須執行此步驟**。

根據本次復盤中使用者提到的主題，從 `400_Knowledge/study/01_processed` 讀取 1-3 張最相關的卡片，給出當天的具體行動建議。

#### 主題 → 卡片對照

| 使用者提到的主題 | 優先讀取的卡片 |
|---|---|
| 焦慮、恐懼、擔憂 | AN1A4, AN1A4C, F6A2A1A5 |
| 情緒失控、衝動 | DU1, DU, A1D1A1 |
| 習慣、行為改變 | A1A1A1A, A1A1A1A1A1, A1A1A1A1A6 |
| 動機不足、拖延 | A1A1A1A7, A1A1A1A1A3 |
| 意志力消耗 | A1A1A1A1A3, A1A1A1A1A3A |
| 人際衝突、溝通卡住 | A1D1A4, A1D1A2A13, A1D1A2A14 |
| 沒有被理解、孤單 | A1D1A2A9A, A1D1A4B |
| 壓力、微壓力 | B1B1A2F3, B1B1A2F4, B1B1A2F5 |
| 自信不足、小挫折 | A1A1G1A2A, A1A1G1A |
| 價值觀衝突、方向感 | B1B1A2F4, DB3D |
| 幸福感、意義感 | AW1C1, DB3F1, B1B1A3E |

#### 行動建議格式

讀完相關卡片後，將以下內容**回覆給使用者，並且同步寫入到今日的日記檔案（300_Journal/YYYY-MM/YYYY-MM-DD.md）的最下方**：

```md
## 💡 今日卡片行動建議

根據今天的復盤，從閱讀筆記中找到這些可以直接用上的洞見：

**1. [建議標題]**
- **卡片來源**：《[書名]》
- **洞見**：[卡片核心概念，一句話]
- **可以怎麼做**：[明天或這週的一個具體行動]

**2. [建議標題]**（若有第二條）
- **卡片來源**：《[書名]》
- **洞見**：[卡片核心概念，一句話]
- **可以怎麼做**：[明天或這週的一個具體行動]
```

**規則**：
- 最多 3 條建議，寧少勿多。
- 每條建議必須連結到卡片中的具體概念，不泛泛而談。
- 行動要具體到可以開始，例如「明天午餐後做一件小事」而非「多放鬆」。
- 建議附上書名，讓使用者知道這是他讀過的東西。
- 若復盤主題沒有明顯對應卡片，可從成長視角給 1 條小成功強化的建議。
