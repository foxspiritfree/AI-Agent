# Listening Communication

運用高共情傾聽與微量認可技術，接住對方的情緒球，降低溝通防衛，建立安全感，讓對話得以流暢深入地進行。

## 使用原則

- **情緒優先，理性在後**：當對方處於情緒中（焦慮、憤怒、挫折）時，任何理性的建議與說教都會被大腦當作威脅。必須先接住情緒，才能開啟理性對話。
- **認可感受，而非認可事實 (Validate Emotion, Not Facts)**：你可以不同意對方的觀點或做法，但你必須無條件認可他們「此時此刻產生的感受是真實且合理的」。
- **接球大於發球**：優秀的傾聽者不急於表現自己或給出正確答案，而是創造一個安全的容器，讓對方願意多說。
- **低防衛姿態**：避免使用防衛性語言（如「但我也是為了你好」、「我沒有這個意思」），改用好奇與探索的姿態面對衝突。

## 快速流程

1. **情緒辨識 (Emotion Recognition)**：
   - 從對方的文字或語氣中，抓出隱含的底層情緒（如：委屈、焦慮、被忽視、害怕失敗）。
2. **施以微量認可 (Micro-Acknowledgment)**：
   - 使用「接球金句」接住情緒。先別給建議，只對感受進行確認與同理（例：「聽起來那時候你一定覺得很孤立無援」、「換作是我，我也會覺得非常生氣」）。
3. **進行搭橋提問 (Bridge Questioning)**：
   - 順著對方剛才說話的最後幾個字、或他最在乎的某個細節，不著痕跡地拋出開放式追問，引導對方說出更多脈絡。
4. **重述與核對 (Paraphrasing)**：
   - 用自己的話簡述對方的處境與感受，向對方核對「我理解得對嗎？」，讓對方產生「我真的被聽見了」的強烈安全感。
5. **輸出對話建議 (Output Options)**：
   - 提供 2-3 個不同程度溫和與共情的對話回覆模板，並附帶解析。

## 傾聽與溝通核心技法

### 1. 微量認可與同理金句 (Micro-Acknowledgment)
當對方拋出負面情緒時，第一時間使用以下句型接球，**嚴禁給出任何建議**：
- **認可委屈**：「這段時間你一個人撐著，真的辛苦了。」
- **認可無力感**：「聽起來你已經做了所有能做的，卻還是遇到這種狀況，一定覺得很挫折。」
- **認可憤怒**：「如果我是你，遇到這種不公平的對待，我也會非常憤怒。」
- **原則**：字句要真誠、具體，指向對方的「感受」，而非評價對方的「行為」。

### 2. 搭橋提問法 (Bridge Questioning)
為了讓對話持續，且不讓對方覺得像在被犯人般審問，順著對方話尾的關鍵字搭建橋樑：
- **技法**：提取對方上一句話的最後一個概念，作為下一句詢問的起點。
- *範例*：
  - *對方*：「這份工作真的讓我感到很**窒息**。」
  - *審問式提問（踩雷）*：「為什麼？你主管對你不好嗎？還是薪水太低？」
  - *搭橋式提問（推薦）*：「你提到『**窒息**』，可以多跟我說說，是哪個瞬間最讓你喘不過氣嗎？」

### 3. 情緒認可的邊界與防雷
- **避免否定句**：不要說「你冷靜一點」、「這沒什麼好生氣的」、「你想太多了」，這是在否定對方的感受真實性。
- **避免搶奪麥克風**：對方在抱怨時，不要說「我以前也是這樣，我跟你說我那時候...」，這會把焦點搶回自己身上。
- **避免過早解決問題**：在對方點頭確認「沒錯，我就是這個意思，你懂我」之前，絕對不要提出任何解決方案。

## 與其他 skill 的邊界

- `strategic-questioning`：`strategic-questioning` 專注於**「主動獲取精確資訊、釐清盲點的問句設計」**；本 skill 則專注於**「被動接收、接住情緒、提供安全感、讓對話得以流暢進行的傾聽與共情技術」**。
- `problem-framing`：當對方的混亂單純是認知或決策層面，不涉及人際衝突與情緒防衛時，直接使用 `problem-framing`。
- `self-brain`：若在溝通中發現使用者特定的溝通偏好或底線，應記錄至 `self-brain` 中以長期沉澱。
