# Strategic Questioning

把「我想知道什麼」轉成對方能回答、願意回答、回答後可行動的問句。

## 使用原則

- 提問前先釐清目的：要理解、驗證、協調、說服，還是建立關係。
- 開場問題要具體、好回答、低壓力。
- 用「如何 / 什麼 / 哪裡卡住」取代容易引發防衛的「為什麼」。
- 先建立信任與共同語境，再問核心問題。
- 輸出可直接複製使用的問句。

## 快速流程

1. 判斷對話目的與對象關係。
2. 分成三層問題：
   - 開場：降低壓力，讓對方開始說。
   - 診斷：取得事實、過往嘗試、限制、動機。
   - 聚焦：確認決策、下一步、承諾或分歧。
3. 檢查問句是否：
   - 範圍夠小。
   - 主體清楚。
   - 對方知道怎麼回答。
   - 不像指責或審問。
4. 產出推薦問法。

## 預設輸出

```markdown
**推薦問法**：[最適合先問的一句]
理由：[一行理由]

**問句組**
- 開場：
- 診斷：
- 聚焦：

**避免這樣問**
- [容易引發防衛或太空泛的問法] -> [改寫]
```

## 常見場景

### 請教高手

- 先交代目標、已嘗試方法、卡住點。
- 問具體下一步，不要求對方代替思考。
- 結尾保留回饋管道，之後回報執行結果。

### 訪談或需求釐清

- 先問過去真實經驗，不先問抽象意見。
- 追問「當時怎麼做」「卡在哪」「最後怎麼解決」。
- 避免引導對方說出你想聽的答案。

### 衝突或分歧

- 先確認共同目標。
- 把立場改寫成可討論的標準或限制。
- 用「我理解的是...有沒有漏掉？」降低對抗感。

### 說服或合作

- 先找對方在意的事實、代價與成功樣貌。
- 保留選擇權，避免命令語氣。
- 把複雜要求拆成小承諾。

## 與其他 skill 的邊界

- `problem-framing`：先釐清自己的問題；需要問別人時再用本 skill。
- `pm`：產品需求文件與範圍定義。
- `self-brain`：把對話結論沉澱成長期脈絡。
