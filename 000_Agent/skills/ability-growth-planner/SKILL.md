# Ability Growth Planner

把「我想變強」拆成可練習、可回饋、可追蹤的能力增長系統。

## 使用原則

- 能力不是願望，要拆成可觀察行為與作品標準。
- 先找最小練習，不先排龐大課表。
- 優先設計回饋迴路：產出、比較、修正、再做。
- 動機不可靠；用環境、提示、預先承諾與降低阻力支撐行動。
- 已能判斷方向時，直接推薦一個訓練主軸。

## 快速流程

1. 定義能力目標：想在什麼情境下做得更好。
2. 拆成能力構件：
   - 知識：需要理解什麼。
   - 技能：需要反覆做什麼。
   - 判斷：需要學會分辨什麼好壞。
   - 表現：最後會產出什麼作品或行為。
3. 設計練習：
   - 逆向工程一個好案例。
   - 抽出 3-5 個可觀察指標。
   - 做一個小作品或小行動。
   - 取得回饋並修正。
4. 設計維持系統：
   - 時間與場景。
   - 行為提示。
   - 降低阻力。
   - 預先承諾。
5. 輸出 7 天或 14 天起步計畫。

## 預設輸出

```markdown
**推薦訓練主軸**：[一個主軸]
理由：[一行理由]

**能力拆解**
- 知識：
- 技能：
- 判斷：
- 表現：

**練習設計**
- 逆向工程對象：
- 追蹤指標：
- 今日最小練習：
- 回饋方式：

**起步計畫**
1. Day 1:
2. Day 2-3:
3. Day 4-7:
```

## 逆向工程方法

- 選一個你真的覺得好的作品、人物行為或案例。
- 拆出表面結構、關鍵決策、細節處理、節奏與限制。
- 把每個要素變成可觀察指標。
- 先臨摹一小段，再做變體。

## 習慣與專注設計

- 不依賴意志力。
- 一次只改一個高影響行為。
- 把行為變小，讓開始成本低於抗拒成本。
- 設定明確提示：時間、地點、前一個行為。
- 用預先承諾處理容易反悔的行動。
- 減少干擾源，尤其是任務切換與即時刺激。

## 回饋指標

優先使用可行動指標：

- 完成次數。
- 作品數量。
- 修正輪數。
- 錯誤類型。
- 他人回饋中的重複訊號。
- 與高手案例的差距。

## 與其他 skill 的邊界

- `journal`：記錄與復盤當天；本 skill 設計訓練系統。
- `notion-task-triage`：排今天做什麼；本 skill 設計能力成長路線。
- `problem-framing`：還不知道問題是什麼時先用它。
- `article-enrichment` / `ak-threads-booster`：特定寫作與 Threads 流程優先用原 skill。
