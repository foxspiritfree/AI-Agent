# Ability Growth Planner

把「我想變強」拆成可練習、可回饋、可追蹤的能力增長系統。

## 使用原則

- 能力不是願望，要拆成可觀察行為與作品標準。
- 先找最小練習，不先排龐大課表。
- 優先設計回饋迴路：產出、比較、修正、再做。
- 動機不可靠；用環境、提示、預先承諾與降低阻力支撐行動。
- 已能判斷方向時，直接推薦一個訓練主軸。

## 快速流程

1. 定義能力目標：想在什麼情境下做得更好。
2. 拆成能力構件：
   - 知識：需要理解什麼。
   - 技能：需要反覆做什麼。
   - 判斷：需要學會分辨什麼好壞。
   - 表現：最後會產出什麼作品或行為。
3. 設計練習：
   - 逆向工程一個好案例。
   - 抽出 3-5 個可觀察指標。
   - 做一個小作品或小行動。
   - 取得回饋並修正。
4. 設計維持系統：
   - 時間與場景。
   - 行為提示。
   - 降低阻力。
   - 預先承諾。
5. 輸出 7 天或 14 天起步計畫。

## 預設輸出

```markdown
**推薦訓練主軸**：[一個主軸]
理由：[一行理由]

**能力拆解**
- 知識：
- 技能：
- 判斷：
- 表現：

**練習設計**
- 逆向工程對象：
- 追蹤指標：
- 今日最小練習：
- 回饋方式：

**起步計畫**
1. Day 1:
2. Day 2-3:
3. Day 4-7:
```

## 逆向工程方法

- 選一個你真的覺得好的作品、人物行為或案例。
- 拆出表面結構、關鍵決策、細節處理、節奏與限制。
- 把每個要素變成可觀察指標。
- 先臨摹一小段，再做變體。

## 習慣與專注設計

- **Fogg 行為模型 (B = MAP)**：
  - 行為 (Behavior) 發生在動機 (Motivation)、能力 (Ability) 與提示 (Prompt) 同時交會的時刻。
  - **動機不可靠**：不要試圖去「提升動機」，因為動機如潮汐，容易疲勞。
  - **能力（簡化行動）是關鍵**：要把行動簡化到「極低成本即可開始」的程度（如：將寫作簡化為「打開 Markdown 並寫下一行字」）。
  - **提示 (Prompt) 分為三類**：
    - *人物提示*：生理或情緒狀態（例：感到疲憊、焦慮）。
    - *情境提示*：環境中的視覺或聽覺訊號（例：桌上放著打開的卡片盒）。
    - *動作提示*：前一個剛做完的固定行為（最推薦，如「當我泡好咖啡坐下時 [動作提示]，我立刻打開寫作軟體 [行為]」）。
- **習慣替代三步驟**：
  1. **確認舊習慣的觸發點 (Prompt) 與獲得的獎勵 (Reward)**：例如「感到壓力/無聊 (Prompt) → 滑社群媒體 (Behavior) → 獲得短暫逃避與刺激 (Reward)」。
  2. **設計「低阻力替代行為」**：必須同樣能帶來大腦需要的放鬆或刺激，但具備建設性。例如「感到無聊 (Prompt) → 抽一張卡片筆記閱讀 (New Behavior) → 獲得思考靈感 (Reward)」。
  3. **微量起步**：新行為必須在 1 分鐘內能完成，藉此建立控制感。
- **拖延症的「大腦保護機制」與因應**：
  - 拖延並非懶惰，而是大腦的「威脅保護機制」在應對壓力（因對任務的焦慮、害怕失敗、完美主義）。大腦的邊緣系統為了保護你免受負面情緒折磨，會主動引導你去進行即時行樂。
  - **因應法（降低大腦防禦）**：
    - *接納與原諒*：接納自己，告訴自己「這很正常，只是大腦在保護我」，研究顯示原諒自己能大幅減少下次拖延。
    - *極致降阻*：對大腦說「我只做兩分鐘，不喜歡隨時可以停下來」，一旦啟動，慣性就會帶領你繼續。
    - *明確極微下一步*：將模糊的「開始寫專案」改為「在 Markdown 檔案中敲下三個標題」。
- 不依賴意志力，一次只改一個高影響行為。
- 用預先承諾（如設定限時、與他人約定）處理容易反悔的行動。
- 減少干擾源，尤其是任務切換與即時刺激。

## 回饋指標

優先使用可行動指標：

- 完成次數。
- 作品數量。
- 修正輪數。
- 錯誤類型。
- 他人回饋中的重複訊號。
- 與高手案例的差距。

## 與其他 skill 的邊界

- `journal`：記錄與復盤當天；本 skill 設計訓練系統。
- `notion-task-triage`：排今天做什麼；本 skill 設計能力成長路線。
- `problem-framing`：還不知道問題是什麼時先用它。
- `article-enrichment` / `ak-threads-booster`：特定寫作與 Threads 流程優先用原 skill。
