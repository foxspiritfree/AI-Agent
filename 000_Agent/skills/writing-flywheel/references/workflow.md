# Writing Flywheel Workflow

## Phase 1: Gather

Read the latest topic candidate report, then skim only the relevant local sources. Prefer narrow lookup over broad reading.

Input sources:

- external Threads topic candidates
- `Inspiration.md`
- study cards
- recent journal notes
- historical Threads posts
- brand voice profile
- `brand_voice.md` Identity Calibration

## Phase 2: Classify

Classify candidates into A/B/C.

The purpose is to reduce user workload:

- A should have personal signal and identity fit, so it is ready to draft.
- B should have identity fit but ask for one small missing personal story or stance.
- C should be parked unless the user can add a strong personal connection or identity bridge.

## Phase 3: Draft

Draft one master version first. Use identity calibration to choose the angle, then keep the actual prose close to the user's own Threads voice.

Do not produce five platform versions before the user edits the master draft.

## Phase 4: Review

After the user edits, run a diagnostic pass:

- Does it still sound like the user?
- Does the angle still fit the user's identity themes?
- Is the claim grounded in experience or notes?
- Is there an obvious AI-tone problem?
- Is there repetition risk against recent topics?

Use AK-Threads-Booster when available for Threads-specific analysis.

## Phase 5: Adapt

Use the multi-platform template after the master draft is settled.

The platform layer changes structure, not the user's belief.

## Phase 6: Learn

When publication data exists, record:

- published platform
- final topic
- source candidate
- performance if available
- what to avoid repeating next week
