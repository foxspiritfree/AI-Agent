# Context Compression

## Summary

Context Compression 是使用者最核心的工作方法之一：把零散背景、流程、判斷與經驗壓縮成 AI、人或未來自己能快速重載的結構。

## Current Understanding

- 使用者對重講背景、重新進入狀態、重建脈絡與重複 onboarding 的成本非常敏感。
- 常見輸出形式包含 spec、rules、AGENTS/GEMINI/CLAUDE 文件、skills、SOP、structured prompts、Notion、Google Sheets 與 Markdown 知識庫。
- 目的不是文檔潔癖，而是降低 re-entry cost、handoff cost、coordination cost 與 cognition loss。
- 此方法直接支撐 `AI-Agent`、self_brain、journal、knowledge automation 與 multi-agent tooling。
