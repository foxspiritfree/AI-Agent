# System Thinking Frameworks

## Summary

使用者反覆使用的總體方法論是：將高摩擦、高脆弱、高依賴的人工作業，轉化為低摩擦、可持續、可擴展的系統。

## Current Understanding

- Root Cause First：從 symptom 往 process、structure、incentive、root cause 下鑽，不停在表層修補。
- Friction Mapping：掃描重複、切換成本、context loss、dependency、資訊流阻塞與 cognitive load。
- Context Compression：把高成本 context 轉成可重用結構，例如 spec、skill files、prompts、SOP、AI rules。
- Abstraction Layering：習慣往上一層抽象，從單點工具看到 workflow、system、infrastructure、cognitive ecosystem。
- Operational Leverage：追求同樣認知成本下更高輸出，將 deterministic work systemize 或 externalize。
- Anti-Fragile Design：重視可恢復、可追蹤、可轉移、可維護，排斥 knowledge trapped in people 與 single point failure。
