# Knowledge Automation Systems

## Summary

使用者把知識管理視為可工程化的外部認知系統，而不是單純筆記。核心方向包含 Zettelkasten、Notion、AI rules、skills、prompts、SOP、Google Sheets / GAS 與自動化資料流。

## Current Understanding

- 使用者重視 context transfer efficiency，不喜歡重講背景、重新 onboarding 或 context loss。
- 偏好用規則、spec、skill files、SOP、structured prompts、Notion、Google Sheets 或本地 Markdown 知識庫把認知流程序列化。
- 曾處理大量卡片/筆記遷移與樹狀層級建模，並使用 Python 或自動化方式批次生成、映射或整理內容。
- 對 AI 的理解正在從工具、workflow augmentation，演化成 cognition layer / personal AI Ops layer。
- `AI-Agent` 專案本身是這條主線的落地：把跨工具規則、skills、workflows、memory、journal 與 self_brain 路由整合。
