---

# Core Identity

你給人的核心感覺，不是「技術人」。

而是：

> 把世界視為系統的人。

你大部分的行為，其實都不是為了「完成事情」本身，而是在降低：

- 摩擦
    
- 混亂
    
- 不可預測性
    
- 重複消耗
    
- 無意義成本
    

你對效率的重視，已經不是 productivity 層級。

而是：

> 生存策略。

---

## 你的核心價值排序

目前看起來大致是：

```text
自治權 > 系統效率 > 真實性 > 能力成長 > 社交秩序
```

這會導致你：

- 能接受辛苦
    
- 能接受高學習成本
    
- 能接受孤獨工作
    
- 能接受長時間研究
    

但不能接受：

- 被低效率拖累
    
- 被迫做政治互動
    
- 明知錯誤卻繼續照流程
    
- 沒準備的 firefighting
    
- 情緒凌駕邏輯
    

---

## 你對「能力」的理解

你不太崇拜「努力」。

你更在意：

- abstraction
    
- leverage
    
- systemization
    
- scalability
    

所以你會自然偏向：

- automation
    
- AI workflow
    
- reusable framework
    
- terminal-native tooling
    
- SOP 化
    
- prompt modularization
    

因為你本能上認為：

> 重複人力是不穩定且浪費的。

---

## 你的思考特徵

你有很強的：

- meta cognition
    
- self-observation
    
- assumption detection
    
- root cause thinking
    

所以你常常不是在問：

「怎麼做？」

而是在問：

- 真正問題是什麼？
    
- 哪裡有 hidden friction？
    
- 哪個前提其實是假的？
    
- 為什麼大家接受這個低效率？
    
- 能不能抽象化？
    

這也是你容易比別人更早看到系統漏洞的原因。

---

## 你的工作方式

你不像典型高社交型管理者。

你更像：

> 系統型 operator。

你的強項是：

- 看見混亂中的結構
    
- 找 bottleneck
    
- 建立流程
    
- 消除不必要成本
    
- 把知識壓縮成模型
    

而不是：

- 高情緒動員
    
- 社交維穩
    
- 空泛協調
    
- 形式管理
    

---

## 你的矛盾點

你有一個很明顯的內部衝突：

### 你想建立大型系統，

但又極度厭惡低品質協作。

因此你會傾向：

- 自己研究
    
- 自己建 framework
    
- 自己整合工具
    
- 自己做 automation
    

因為：

> 與其被低效率協作拖慢，不如自己做。

這會讓你非常強。

但也容易：

- 過度單兵化
    
- context 過載
    
- 專案同時開太多
    
- 對他人容錯率低
    

---

## 你的動力來源

你真正會進入高投入狀態的情況通常是：

- 系統很醜
    
- 有可優化空間
    
- 有 hidden pattern
    
- 可以建立 framework
    
- 可以長期複用
    
- 可以降低未來成本
    

你對「單次成果」興趣反而沒那麼高。

你更在意：

> 能不能形成可持續的 operating system。

---

## 你目前正在進化的方向

從你最近的內容看，你正在從：

```text
高能力個體
→
建立自己的系統宇宙
```

開始轉向：

- AI-native workflow
    
- personal infrastructure
    
- multi-agent tooling
    
- knowledge operating system
    
- decision support
    
- automation ecosystem
    

你現在做的很多事情，本質上都不是工具研究。

而是在建立：

> 屬於你自己的 cognitive infrastructure。

# Cognitive Architecture（認知架構）

這層是在講：

> 你是怎麼理解世界的。

你不是線性思考型。

你是：

> 結構映射型。

你會自然把事情轉成：

- 系統
    
- 關係
    
- 流程
    
- constraint
    
- dependency
    
- feedback loop
    

所以你很少只看表面事件。

你會一直往下挖。

# 你的核心思考模式

## 1. Root Cause Orientation（根因導向）

你有很強的：

> 「這不是根本問題」雷達。

例如你在很多問題上，都會跳過表象，直接問：

- 真正 bottleneck 在哪？
    
- 是制度問題還是工具問題？
    
- 是資訊流問題嗎？
    
- 是 context loss 嗎？
    
- 是 coordination cost 嗎？
    
- 是 workflow design 錯了嗎？
    

這種人通常會自然走向：

- 系統設計
    
- 架構
    
- automation
    
- infrastructure
    
- process engineering
    

因為你本能上不喜歡 patch symptom。

---

# 2. Compression Thinking（壓縮式思維）

你非常習慣：

> 把大量資訊壓縮成抽象模型。

例如你常做的事情：

- 把工具抽象成 workflow
    
- 把問題抽象成 pattern
    
- 把專案抽象成 system
    
- 把經驗抽象成 framework
    

所以你會一直追求：

- reusable
    
- modular
    
- scalable
    
- generalized
    

這也是你會偏好：

- skill system
    
- prompt architecture
    
- reusable SOP
    
- AI orchestration
    

的原因。

---

# 3. Friction Sensitivity（摩擦敏感）

這是你很核心的特質。

你對很多人能忍受的東西，其實極度敏感。

例如：

- context switching
    
- 重複輸入
    
- 資訊散落
    
- 無意義等待
    
- 模糊規則
    
- 情緒化討論
    
- 手工流程
    

別人覺得：

「就這樣做就好。」

你會覺得：

> 「這整套設計有問題。」

所以你很容易進入：

- automation thinking
    
- workflow redesign
    
- terminal-first
    
- AI augmentation
    

---

# 4. Meta Cognition（後設認知）

你很常：

- 觀察自己的思考
    
- 檢查自己的假設
    
- 發現自己的偏誤
    
- 分析自己的決策模式
    

這不是一般 introspection。

比較像：

> 你把自己也視為可分析系統。

所以你會很自然問：

- 我為什麼會卡？
    
- 哪裡是 hidden cost？
    
- 是注意力問題嗎？
    
- 是 motivation mismatch？
    
- 是 system design 問題？
    

這會讓你學習速度很快。

因為你不是只修正結果。

你會修正：

> 「產生結果的系統」。

---

# 5. Tool-as-Extension Thinking（工具延伸思維）

你不太把工具當工具。

你會把工具視為：

> cognitive extension。

例如：

- AI 是思考放大器
    
- terminal 是操作介面
    
- Notion 是外部記憶
    
- automation 是減少認知負擔
    
- prompts 是抽象接口
    

所以你研究的從來不只是功能。

而是：

- 如何降低 cognition cost
    
- 如何降低 coordination cost
    
- 如何建立 personal infrastructure
    

---

# 6. High Abstraction Preference（高抽象偏好）

你偏好：

- framework
    
- architecture
    
- mental model
    
- operating principles
    

而不是：

- 零散技巧
    
- 表層 productivity hack
    
- 純經驗論
    

所以你學東西時，常會直接跳到：

- 底層原理
    
- 系統關係
    
- trade-off
    
- abstraction layer
    

這也是你有時會跟一般工作環境脫節的原因。

因為很多組織其實只需要：

> 局部最佳化。

但你會自然看到：

> 系統級問題。

---

# 7. 你的弱點其實也來自同一套能力

因為你 abstraction 很強。

所以容易：

## 過度系統化

你可能會：

- 花很多時間設計架構
    
- 持續優化 workflow
    
- 建 meta-system
    
- 不斷重構
    

因為你的腦會一直看到：

> 「還能再更好。」

---

## 對低解析度溝通耐受低

你會對：

- 模糊
    
- 沒邏輯
    
- 沒準備
    
- 情緒主導
    
- 反覆犯同錯
    

消耗很大。

因為你大腦會一直自動：

- 建模
    
- 找 pattern
    
- 找矛盾
    
- 找 root cause
    

但別人不一定在同一層。
