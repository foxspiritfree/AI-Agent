# Writing Flywheel Source Routing

## Weekly Packet

Use sources in this order:

1. `../../inbox/Inspiration.md`
2. `../../../400_Knowledge/self_brain/`
3. `threads_daily_tracker.json`
4. `posts_by_date.md`
5. `brand_voice.md`
6. `../../../400_Knowledge/study/config/classification_map.md`
7. `../../../400_Knowledge/body/config/classification_map.md`
8. `../../../400_Knowledge/study/01_processed/`
9. `../../../400_Knowledge/body/01_processed/`
10. Recent `../../../300_Journal/YYYY-MM/*.md`
11. `data/reports/topic-candidates-YYYY-MM-DD.md`
12. `data/reports/topic-candidates-ai-YYYY-MM-DD.md`

Rule:

- `Inspiration.md` generates candidates
- `self_brain`, prior posts, brand voice, and journal notes decide whether the candidate has enough personal signal to draft
- use `study/config/classification_map.md` and `body/config/classification_map.md` first to identify likely branches
- card-box references must come from existing files in `study/01_processed` or `body/01_processed`
- do not cite the map itself as a card; use it only to locate actual cards
- external sources validate freshness and angle
- if personal experience exists, produce one draft-ready item; a weekly packet may include several draft-ready items
- if personal experience is missing, produce a precise replenishment request
- if no existing study/body card matches, write `з„Ўж—ўжңүеҚЎзүҮе‘Ҫдёӯ`; optional generated note text must be labeled `еҸҜжІүжҫұжҲҗж–°еҚЎзҡ„зӯҶиЁҳиҚүжЎҲ`

## User Reply Intake

Use:

1. latest weekly packet item the user selected
2. pasted user reply or `data/user-replies/YYYY-MM-DD.md`
3. `brand_voice.md`
4. nearest prior posts from `posts_by_date.md`

## Master Draft

Use:

1. selected packet item
2. user reply
3. `brand_voice.md`
4. `compiled/voice_fingerprint.md` when available
5. closest study or journal support

## Platform Adaptation

Use:

1. user-revised master draft
2. `one_draft_multi_platform_template.md`

Do not adapt a draft that still lacks the user's real turn.
