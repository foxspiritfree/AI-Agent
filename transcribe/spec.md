# 修正版 Spec：YouTube Full Caption to Markdown Extension

## 1. 產品目標

建立一個自用 Chrome Extension，在 YouTube 影片頁面中：

1. 一次抓取整部影片的字幕。
    
2. 轉成 Markdown。
    
3. 一鍵複製。
    
4. 若抓不到字幕，再切換到語音轉文字模式。
    

---

## 2. 核心策略調整

原本 spec 的「監聽畫面字幕」要降級成備援方案。

新的優先順序：

|優先級|方法|說明|是否 MVP|
|--:|---|---|---|
|1|從 YouTube player response 找 caption track|一次取得完整字幕|是|
|2|從 transcript panel 抓完整逐字稿|備援|是|
|3|監聽畫面字幕|最後備援，只能邊播邊抓|否|
|4|語音轉文字|沒字幕時使用|Phase 2|

YouTube 官方 Data API 的 `captions.download` 雖然可下載字幕，但要求使用者有影片編輯權限，因此不適合拿來抓一般公開影片字幕。這個 extension 應該從目前頁面的 player data / caption track 資訊取得字幕，而不是走 YouTube Data API。([Google for Developers](https://developers.google.com/youtube/v3/docs/captions/download?utm_source=chatgpt.com "Captions: download | YouTube Data API"))

# 4. 關鍵技術判斷

Chrome content script 可以讀 DOM，但它跑在 isolated world，不能直接讀取 YouTube 頁面 JS 裡的變數。([Chrome for Developers](https://developer.chrome.com/docs/extensions/develop/concepts/content-scripts?utm_source=chatgpt.com "Content scripts | Chrome for Developers"))  
所以要取得 YouTube player response，有兩條路：

|路線|說明|建議|
|---|---|---|
|DOM script tag parsing|從頁面 HTML / script 文字中解析 `ytInitialPlayerResponse`|先做|
|MAIN world injection|用 `chrome.scripting.executeScript({ world: "MAIN" })` 進入頁面主世界讀取 player data|備援|

Chrome `scripting.executeScript` 支援 `MAIN` world，代表 extension 可以在頁面主 JS 環境執行注入腳本，而不是只在 isolated world。([Chrome for Developers](https://developer.chrome.com/docs/extensions/reference/api/scripting?utm_source=chatgpt.com "chrome.scripting | API - Chrome for Developers"))

# 9. Caption Track Finder Spec

## 目標

從 YouTube 頁面取得：

```ts
playerResponse.captions.playerCaptionsTracklistRenderer.captionTracks
```

## 主要策略

### Strategy A：解析 script 內的 `ytInitialPlayerResponse`

```ts
function extractPlayerResponseFromScripts(): unknown | null {
  const scripts = Array.from(document.querySelectorAll("script"));

  for (const script of scripts) {
    const text = script.textContent || "";

    const marker = "ytInitialPlayerResponse = ";
    const index = text.indexOf(marker);

    if (index === -1) continue;

    const jsonStart = index + marker.length;
    const jsonEnd = text.indexOf(";</script>", jsonStart);

    // 實作時不要只靠這個，應寫 bracket matching
  }

  return null;
}
```

實作時不要用單純 regex 硬切 JSON，建議寫一個 brace matching parser。

---

### Strategy B：注入 MAIN world 讀取 player response

```ts
async function getPlayerResponseFromMainWorld() {
  const [result] = await chrome.scripting.executeScript({
    target: { tabId },
    world: "MAIN",
    func: () => {
      // 依 YouTube 實際可用變數與 player API 嘗試取得
      return window.ytInitialPlayerResponse || null;
    }
  });

  return result.result;
}
```

# 10. Language Selector Spec

## 預設選擇順序

```ts
const PREFERRED_LANGUAGES = [
  "zh-TW",
  "zh-Hant",
  "zh",
  "en",
  "ja"
];
```

## 選擇規則

1. 若有手動字幕，優先於自動字幕。
    
2. 語言優先序：繁中 → 中文 → 英文 → 日文 → 第一個 track。
    
3. 若只有 auto-generated caption，也可以使用。
    
4. Popup 顯示可切換語言。
    

```ts
function selectBestTrack(tracks: CaptionTrack[]): CaptionTrack | null {
  const manual = tracks.filter(t => !t.isAutoGenerated);
  const auto = tracks.filter(t => t.isAutoGenerated);

  const candidates = [...manual, ...auto];

  for (const lang of PREFERRED_LANGUAGES) {
    const found = candidates.find(t => t.languageCode === lang);
    if (found) return found;
  }

  return candidates[0] ?? null;
}
```

# 11. Caption Fetcher Spec

## 目標

拿到 `CaptionTrack.baseUrl` 後，fetch 完整字幕。

```ts
async function fetchCaptionTrack(track: CaptionTrack): Promise<string> {
  const url = new URL(track.baseUrl);

  // 優先要求 JSON3，若失敗再 fallback XML
  url.searchParams.set("fmt", "json3");

  const res = await fetch(url.toString());

  if (!res.ok) {
    throw new Error(`Failed to fetch caption track: ${res.status}`);
  }

  return await res.text();
}
```

## Fallback

```txt
1. fmt=json3
2. 原始 baseUrl
3. fmt=srv3
4. fmt=vtt
```

# 12. Caption Parser Spec

## 支援格式

|格式|優先級|說明|
|---|--:|---|
|json3|1|最好 parse|
|XML / srv3|2|YouTube 常見字幕格式|
|VTT|3|備援|

---

## JSON3 Parser

```ts
function parseJson3(raw: string): TranscriptSegment[] {
  const data = JSON.parse(raw);

  return data.events
    .filter((event: any) => event.segs?.length)
    .map((event: any) => ({
      startSec: event.tStartMs / 1000,
      durationSec: event.dDurationMs ? event.dDurationMs / 1000 : null,
      text: event.segs.map((seg: any) => seg.utf8).join("").trim(),
      source: "caption-track" as const
    }))
    .filter((segment: TranscriptSegment) => segment.text);
}
```

---

## XML Parser

```ts
function parseXmlCaption(raw: string): TranscriptSegment[] {
  const doc = new DOMParser().parseFromString(raw, "text/xml");
  const nodes = Array.from(doc.querySelectorAll("text"));

  return nodes.map(node => ({
    startSec: Number(node.getAttribute("start") || 0),
    durationSec: Number(node.getAttribute("dur") || 0),
    text: decodeHtmlEntities(node.textContent || "").trim(),
    source: "caption-track" as const
  })).filter(segment => segment.text);
}
```

# 15. Error Handling

|狀況|UI 顯示|下一步|
|---|---|---|
|找不到 player response|找不到 YouTube 播放資料|嘗試開啟逐字稿面板|
|找不到 captionTracks|這部影片沒有可取得字幕|顯示 STT 選項|
|fetch caption 失敗|字幕抓取失敗|嘗試其他格式|
|parse 失敗|字幕格式解析失敗|顯示 raw fallback|
|clipboard 失敗|複製失敗|改用下載 `.md`|
|影片有限制|無法取得字幕|改用 STT|

## 最終建議

你要的其實是：

> **Full caption track extractor，不是 caption observer。**

所以 MVP 應該直接做「從 player response 找 captionTracks → fetch baseUrl → parse → Markdown」。  
語音轉文字等 MVP 成功後再加。
